Detailed Description

Frame information.

This structure provides information about a movie frame.

Member Data Documentation

int OmFrameInfo::anchorFrame

The anchor frame number for this frame.

Intracoded material is self-anchored (i.e. anchor = frame). For non-intracoded material the anchor frame points to the previous I or P frame. Set to 0 when writing a frame.

enum OmSampleCodingType OmFrameInfo::codingType

Sample coding type.

int OmFrameInfo::displayOffset

The time offset (in frames) from file order to display order for this frame.

This field can be 0, positive or negative depending on the display position of the frame. A positive offset indicates that this frame's display time is in the future. A negative display offset indicates that the frame's display time is in the past.

bool OmFrameInfo::displayOffsetAvailable

True when the displayOffset field contains a valid display offset.

False if there is no frame reordering information available for the track.

OmMediaId OmFrameInfo::mediaId

Id of the file where this frame is stored.

uint OmFrameInfo::nChunks

Number of chunks for this frame.

For video this is always 1, for audio it can be 1 or 2. When writing a frame it should always be 1.

uint OmFrameInfo::nSamples

Number of samples in this frame.

uint_64 OmFrameInfo::offset0

File offset of the first chunk.

When writing a frame in a referenced movie this should be the offset of the frame, otherwise it should be set to 0.

uint_64 OmFrameInfo::offset1

File offset of the second chunk.
